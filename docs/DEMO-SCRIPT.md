# AccessCrash 2:50 Demo Script

## Recording target

- **Hard limit:** under 3:00
- **Target:** 2:40–2:50, leaving recovery margin
- **Format:** 16:9, 1080p, audible English narration, captions, readable cursor
- **Build:** the exact public build submitted to judges
- **Data:** only fictional Pineglass Institute · Access Grant content
- **Core story:** instructions → GPT draft → human confirmation → deterministic
  access crash → synthetic minimum repair set → reachable regression proof

Keep all primary text inside a safe area with at least 6% horizontal margin.
Before recording, inspect the first and last frame at normal YouTube playback
size; no title, subtitle, label, or caption may start against the video edge.

Do not show private tabs, email addresses, account menus, API dashboards,
environment files, identity headers, notifications, or secrets.

## Shot list and narration

### 0:00–0:14 — The problem

**Screen:** AccessCrash hero and the line **Eligibility is not access**.

**Narration:**

> A student can qualify for help and still have no workable path to apply.
> AccessCrash finds where an education process becomes unreachable before a
> real person pays the price.

### 0:14–0:30 — Load a safe synthetic process

**Action:** Load **Pineglass Institute · Access Grant**. Point to the fictional,
synthetic, no-real-student-data, and no-persistence labels.

**Narration:**

> This is a fictional emergency-grant process. There are no real students and
> nothing is saved by AccessCrash. The product can also accept bounded public or
> organization-owned non-personal process instructions, but never student
> records. The bundled case keeps this demonstration reproducible.

### 0:30–0:55 — GPT-5.6 compiles, but does not decide

**Action:** Select **Compile access path**. Show the live model label, cited
rule rows, and unconfirmed state.

**Narration:**

> GPT-5.6 converts the instructions into a structured process graph. Each rule
> stays attached to the source text that supports it. Notice the state: draft,
> source cited, not confirmed. The model has no reachability or eligibility
> verdict in its output contract.

If the recorded run is fallback, replace “GPT-5.6 converts” with:

> The live model is unavailable, so AccessCrash clearly switches to its bundled
> synthetic fallback. It does not pretend this graph came from the pasted
> source.

Do not record fallback while narrating it as live.

### 0:55–1:16 — Human confirmation boundary

**Action:** Open two citations, confirm the visible rules, then select
**Confirm N source-linked rules**.

**Narration:**

> A person checks the graph before code is allowed to judge it. Confirmation
> approves this representation of the synthetic instructions; it does not
> certify the policy as fair, legal, or complete.

### 1:16–1:43 — Crash-test a capability twin

**Action:** Show the fixed standard profile and the capability-constrained twin.
Select **Run deterministic crash test**.

**Narration:**

> Eligibility is held constant as an external synthetic assumption; AccessCrash
> does not evaluate it. Only access capabilities differ. The standard path
> reaches submission. The constrained twin loses its final route. Deterministic
> code, not GPT, returns blocked: no reachable path.

### 1:43–2:03 — Inspect the exact failure

**Screen:** BEFORE `BLOCKED`, the process map, source-linked blocker, and any
relevant cycle or unavailable channel.

**Narration:**

> This is not a generic risk score. AccessCrash shows the exact transition that
> failed, the source rule behind it, and the graph evidence proving that no
> alternate route remains. If the evidence were incomplete, the result would be
> unknown instead of a guess.

### 2:03–2:26 — Test the minimum repair set

**Action:** Select **Test 3-change repair set**. Show AFTER `REACHABLE` and
the BEFORE / AFTER comparison.

**Narration:**

> Now we test the minimum demo repair set: email verification, mobile upload,
> and evening review — three bounded alternatives applied together in memory.
> AccessCrash does not edit a real program. The same deterministic engine
> recomputes the same capability profile, and the path reopens.

### 2:26–2:44 — Implementation proof and role split

**Screen:** Prepared, non-sensitive view of the architecture and verified test
commands, then return to the result.

**Narration:**

> Codex helped design, build, audit, test, and document the product. GPT-5.6 has
> one runtime job: compile source-grounded drafts. Human confirmation controls
> interpretation, and deterministic graph code owns every result.

Only show a check as passed if it succeeded on the exact recorded source state.

### 2:44–2:50 — Close

**Screen:** Product name, tagline, public source, and live URL if verified.

**Narration:**

> AccessCrash. Eligibility is not access. Test the path before people crash into
> it.

source-grounded process graph, requires human confirmation, and then uses
deterministic code to prove whether a bounded capability profile can reach the

does not modify a real program.
